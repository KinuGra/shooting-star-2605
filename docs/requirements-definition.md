**プロダクト名（仮）**: プログラミング課題採点DX化アプリ
**ステータス**: 企画決定 / MVP定義

## 1. 概要

大学のプログラミング授業における、課題採点業務をDX化するWebアプリ。学生が提出したコードをジャッジサーバーで自動採点し、教員・TAの採点負担を削減する。

## 2. 背景・課題

- 大学のような法人のDX化が遅れているという社会課題を解決する。
- 現状、教授が手動でコードを読んだり、TAが動作確認してハンコを押しており非効率。
- これを効率化し、教員・TAの負担を減らす。

## 3. ターゲットユーザー

| ロール | 説明                                                     |
| ------ | -------------------------------------------------------- |
| 教員   | 授業・課題の管理、採点の確認・修正、学生の課題にコメント |
| TA     | 採点補助、提出物の確認                                   |
| 大学生 | 課題の提出、成績の確認                                   |

## 4. 利用シーン

- プログラミングの授業
- アプリケーション開発の授業
- データ構造・アルゴリズムの授業

## 5. ソリューション方針

ジャッジサーバーを設置し、学生がコードをWebアプリ上で提出。自動採点を行い、教員が必要に応じて修正・コメントできる仕組みとする。

---

## 6. 機能要件

### 6.1 学生向け機能

| 機能                    | 詳細                                                                                                          |
| ----------------------- | ------------------------------------------------------------------------------------------------------------- |
| 生徒作成                | 学生アカウントの作成                                                                                          |
| 授業参加                | 教師が作成した招待コードを入力して授業に参加                                                                  |
| 課題一覧表示            | 参加している授業の課題一覧を表示                                                                              |
| 課題提出                | コードを提出。複数回提出可能。課題の期限終了時、最終的な提出結果のみが採点される                              |
| 採点                    | AtCoderのように即時採点。その後、教師が修正・コメント可能                                                     |
| 採点結果一覧 / 成績一覧 | 成績は授業ごとに、各課題の最後の提出の点数の和 / 最大点。採点結果は課題一覧画面、成績一覧は授業一覧画面に表示 |
| 各提出詳細              | 点数、状態、コード、コメントを表示                                                                            |

**学生の画面遷移フロー**

生徒作成 → 授業参加 → 課題一覧表示 → 課題提出 → 採点 → 採点結果一覧・成績一覧 → 各提出詳細

### 6.2 オーナー・教師向け機能

| 機能                 | 詳細                                                               |
| -------------------- | ------------------------------------------------------------------ |
| 教師作成             | オーナーが教師を作成（教師名を入力）                               |
| 授業作成             | 教師が授業を作成（授業名を入力）                                   |
| 課題作成             | 課題名、言語名、課題の内容、テストケース、期限、公開・非公開を設定 |
| 提出済み課題一覧表示 | 生徒ごとに表示。採点済み・未採点のフィルター機能                   |
| 提出詳細             | 学生と同じ内容が閲覧可能 + 採点修正・コメントが可能                |

**オーナー・教師の画面遷移フロー**

オーナーが教師作成 → 教師が授業作成 → 課題作成 → 提出済みの課題一覧表示 → 提出詳細

---

## 7. 非機能・採点の前提

- 自動採点はジャッジサーバー上で実行。
- 提出は複数回可能だが、採点対象は期限終了時点の最終提出。
- 成績集計は「各課題の最後の提出点数の和 / 満点」で算出。
