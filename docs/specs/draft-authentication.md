# 認証・認可仕様

## 認証方式

メールアドレス＋パスワード＋TOTP（2FA）による認証。

| 項目 | 内容 |
|------|------|
| 認証方式 | メール＋パスワード |
| 二要素認証 | TOTP（RFC 6238）、任意設定 |
| セッション管理 | JWT（Bearer Token） |
| パスワードハッシュ | Argon2id |

---

## TOTP（2FA）仕様

### 概要

- TOTP は任意設定（オプトイン）
- Google Authenticator / Authy などの認証アプリに対応
- 有効化後のログインは パスワード認証 → TOTP コード入力 の2ステップ

### 有効化フロー

```
1. ユーザーが設定画面で「2FAを有効化」を選択
2. サーバーが TOTP シークレットを生成し、QRコードを表示
3. ユーザーが認証アプリでQRコードをスキャン
4. ユーザーが認証アプリに表示された6桁コードを入力して確認
5. 確認成功後、バックアップコード（8件）を表示・保存
6. 以降のログインで TOTP コードが必要になる
```

### 無効化フロー

```
1. ユーザーが設定画面で「2FAを無効化」を選択
2. 現在の TOTP コードを入力して確認
3. シークレット・バックアップコードをすべて削除
```

### ログインフロー（2FA有効時）

```
1. メール＋パスワードで認証
2. 2FA が有効な場合、TOTP コード入力を要求
3. 6桁コード（または バックアップコード）を入力
4. 検証成功後、JWT を発行
```

### バックアップコード

- 有効化時に8件生成（ランダムな英数字）
- 1回使用すると無効になる（使い捨て）
- 設定画面から再生成可能（再生成時に旧コードはすべて無効）

## 登録フロー

### 学生（STUDENT）の自己登録

```
1. メールアドレス・パスワードを入力して登録
2. メール認証リンクをクリックして有効化
3. ログイン可能になる
```

### 教師（TEACHER）アカウントの発行

```
1. ADMIN が管理画面からメールアドレスを入力して発行
2. 仮パスワードが記載されたメールが送信される
3. 教師が初回ログイン後にパスワードを変更する
```

---

## 授業への参加フロー

```
1. 教師が授業コード（ランダム文字列）を学生に共有する
2. 学生がログイン後、授業コードを入力して参加
3. 授業内ロール STUDENT として登録される
```

### 授業コード

- ランダムな英数字（例: `A3F9KZ`）
- 教師はいつでも再発行（ローテーション）可能
- 再発行後、古いコードは無効になる

---

## メール通知

以下のイベントで通知メールを送信する。

| イベント | 送信先 |
|----------|--------|
| 授業への招待（コード共有） | - （コードを直接共有する形のため通知なし） |
| 課題の締め切り前リマインダー | 提出していない受講生 |
| 課題の返却 | 受講生全員 |
| 教師アカウント発行 | 発行された教師 |

### リマインダーの送信タイミング

- 締め切りの 24 時間前

---

## 認可ルール

- JWT のペイロードにグローバルロール（`global_role`）を含む
- 授業内ロールは DB から取得して検証する
- 返却前の提出結果・フィードバックは本人でも閲覧不可
- 未公開の課題は受講生・TA に対して非表示
- `publish_at` が未来日時の課題は、その日時まで非表示
